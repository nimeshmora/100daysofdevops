Day 1: Introduction to DevOps

What is DevOps? DevOps is a set of practices that combines software development and IT operations.

It focuses on continuous development, integration, and deployment to improve software delivery.

Day 2: Introduction to Linux

Why Linux? Linux is widely used in DevOps environments for its flexibility and performance.

Basic Linux commands: ls, cd, mkdir, rm, cp, mv, touch, cat, echo.

Day 8: Git Basics

Version Control: Git helps track changes in code over time.

Basic commands: git init, git add, git commit, git push, git pull.

Day 9: Working with GitHub

GitHub is a remote repository hosting service for collaborative coding.

Commands to interact with remote repositories: git clone, git fetch, git merge.

Day 15: CI/CD Basics

CI/CD Pipeline: Automates software build, test, and deployment.

Popular tools: Jenkins, GitHub Actions, GitLab CI/CD.

Day 16: Pipeline Stages

Stages: Source code management, build automation, testing, deployment.

Day 22: Introduction to Docker

What are Containers? Containers are lightweight, portable environments for running applications.

Day 23: Running Containers

Commands: docker run, docker ps, docker stop, docker rm.
